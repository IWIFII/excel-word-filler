## excel-word-filler



### 核心思路

- **按“单元格”和“行”操作，不做全文替换**  
  使用 `python-docx` 逐格写入表格内容，而不是在 XML 里搜字串或用 Word 的查找替换，避免：
  - 占位符被拆成多个 run，替换一半的情况；
  - 破坏段落/域/编号/合并单元格结构。

- **复制模板中的“样例行”，继承完整样式**  
  Word 模板中保留一行你满意的“样例数据行”（字体、行高、边框、对齐都设置好）。  
  程序通过 `deepcopy` 复制这一行的底层 XML，再只替换单元格文本，因此：
  - 所有数据行的样式与样例行完全一致；
  - 不需要在代码里硬编码任何样式。

- **自动按“列名”对齐 Excel 和 Word**  
  默认开启 `AUTO_MATCH_HEADERS`：
  - 从 **Excel 第一行**读取真实表头；
  - 从 **Word 表格表头行**读取每一列表头文字；
  - 经过规范化（去多余空格、统一全角/半角括号和 Unicode 变体）后按名称一一匹配；
  - 最终按 **Word 表头的顺序**选取和排列 Excel 列。  
  这意味着：
  - 你只需在 Word 表头里写出想要的列名（例如“游客编号 / 游玩日期 / 消费金额（元）”），
  - 程序会自动从 Excel 中找到对应列并按顺序填入。


### 使用方式

1. 将 Excel 文件放在 `source_docs/`（或项目根目录）下，将 Word 模板放在 `templates/`（或根目录）下。  
2. 在模板目标表格中：
   - 第 `HEADER_ROW_INDEX` 行写好表头文字（需与 Excel 表头“语义一致”即可）；
   - 第 `TEMPLATE_DATA_ROW_INDEX` 行保留一行样例数据行作为样式模板。  
3. 激活虚拟环境并运行：

```bash
python fill_table.py
```

生成的成品 Word 在 `output/filled.docx` 中。
